# RoleForge — Principes produit

1. Le héros avant l’IA.
2. L’interface doit donner envie de jouer.
3. Le texte ne doit jamais être la seule interaction.
4. Chaque décision importante doit être sauvegardée.
5. L’app doit fonctionner sans clé API grâce au mode test.
6. Chaque fonctionnalité doit pouvoir évoluer vers un vrai système RPG.
7. On spécifie avant de coder.
8. On garde la V1 étroite et jouable.
